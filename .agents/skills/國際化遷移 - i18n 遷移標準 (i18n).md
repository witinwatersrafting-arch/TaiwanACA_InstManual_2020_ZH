# 國際化遷移 - i18n 遷移標準 (i18n Migration)

本手冊用於規範 Astro 組件與頁面從單一語言到多語言 (i18n) 架構的遷移流程與技術標準。

## 1. 核心迁移邏輯 (Core Logic)

所有組件必須透過 `lang` Prop 判定語系，並使用 `src/utils/i18n.ts` 提供的工具函數處理內容與路徑。

- **`t(key)`**: 內容翻譯 (Text Translation)
- **`l(path)`**: 路徑區域化 (Localized Link Generation - 自動為 `zh` 加上 `/zh/` 前綴)

## 2. 操作 SOP

1.  **內容提取**: 將組件與外部 JS 數據中的硬編碼文字移至 `src/utils/i18n.ts` (翻譯對象) 或 `src/data/` (數據對象)。
2.  **Props 注入**:
    ```astro
    interface Props { lang?: 'en' | 'zh'; }
    const { lang = 'en' } = Astro.props;
    const { t, l } = useI18n(lang);
    ```
3.  **無障礙 (A11y)**: 所有 `aria-label`, `alt`, `title` 必須對應語系翻譯。
4.  **JS 數據隔離**: 如果 Script 中有內容數據，應使用 `define:vars` 由 Astro 傳入編譯後的翻譯數據，避免在 Client-side 進行重複翻譯與路徑判定。

## 3. SEO 規範

- **統一入口**: 使用 `l(path)` 確保內部靜態連結在 `/zh/` 目錄下自動路由至對應區域版本。
- **HTML `lang` 屬性**: 確保根部 `Layout` 的 `lang` 屬性被正確傳遞。

## 4. 避免 React 錯亂

嚴禁在 `.astro` 檔案 frontmatter 中嘗試導入 React-context 或 Hook。所有國際化上下文應透過 props 層層傳遞，保持單一數據流。
