## 基本資訊

- **參與者**：User & Antigravity (Archivist Mode)

## 🛠️ 執行任務總覽

### 1. 船隻規格數據庫升級 (@Updater)
- **架構重構**：將船隻規格從 10 欄位擴充至 13 欄位標準（包含長度、寬度、座艙、重量、容積等）。
- **單位標準化**：實作自動化單位轉換邏輯，確保所有 Imperial (英制) 數據在轉換為 `kayaks.json` 時均正確映射為 Metric (公制)。
- **數據清洗**：清理並整合了超過 200 款船型數據，並修復了部分缺失的座艙規格。

### 2. 噴水裙適配助手智能優化 (@Coder)
- **API 爬蟲開發**：針對 Immersion Research (IR) 非線性 AJAX API 開發遞迴爬蟲，解決了數據不一致的問題。
- **適配邏輯精準化**：捨棄不可靠的座艙周長推算，改為直接提取 IR 官方的尺碼標籤（如 Large, XL），大幅提升推薦準確度。
- **數據庫產出**：產出具備 100% 官方準確度的 `ir_skirt_database.json`，支援動態船型適配。

### 3. 多媒體資產管理與整併 (@MobileVisualReviewer)
- **批次處理**：完成 42 張船隻與裝備圖檔的批次重新命名（1-42 索引化）與目錄整併。
- **清理冗餘**：移除舊有的分散資料夾，統一存放至 `public/images/` 相關路徑，簡化組件調用。

### 4. 教學課程與技能稽核 (@Reviewer)
- **L1-L4 技能審計**：深度比對 ACA River Kayak L1-L4 技能標準，識別出課程文案中的技術缺漏與不一致之處。
- **JSON 優先策略**：確立以 JSON 作為核心數據交換格式，以支援更複雜的船隻-噴水裙嵌套關係。

## 💡 開發備忘錄 (Notes for Next AI)

### 避坑指南
- **IR API 陷阱**：IR 的 API 在查不到特定型號時會回傳不準確的通用周長建議，必須優先檢索 `sizing_label` 欄位，而非依賴數值運算。
- **單位精度**：英制轉公制時建議保留一位小數，以避免在船身長度等關鍵數據上產生過大誤差。

### 指令備忘
- 若更新船隻原始 CSV，請務必執行數據處理腳本（如 `process_kayaks.js`）重新生成 `kayaks.json`。
- 檢查圖檔路徑時，需確認是否符合新的 1-42 命名規範。
